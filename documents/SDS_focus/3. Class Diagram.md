# **3. Class Diagram**

---

본 프로젝트는 **Spring Boot 기반의 MVC 아키텍처**를 적용하였으며,  
데이터베이스의 논리 구조(**ERD**)를 기반으로 객체지향적 클래스로 확장하였다.  
클래스는 **도메인(Domain)**, **서비스(Service)**, **컨트롤러(Controller)**로 구성된다.  
이번 장에서는 다양한 관점에서 바라본 **Class Diagram**과  
각각에 대한 설명을 기술한다.

---

### **본 문서의 Class Diagram 목차**

- **3.1 공통 단일 클래스**  
  : 여러 서비스 도메인에서 공통으로 사용하는 단일 클래스의 역할과 속성을 자세하게 설명한다.

- **3.2 클래스 다이어그램 (Domain별)**  
  : 앞에서 제시한 ERD를 바탕으로 각 도메인별 클래스 간의 관계를 나타내며 설명한다.

- **3.3 DTO 클래스**  
  : DTO 클래스는 데이터 전송을 위한 객체로,  
    비즈니스 로직을 포함하지 않고 데이터를 캡슐화하여  
    계층 간, 시스템 간 데이터 전송을 최적화한다.  
    웹과 통신하기 위해 필요한 DTO의 속성과 기능에 대해 정의한다.
